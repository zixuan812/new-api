# v0.0.0

## Changed

- `web/default/src/components/layout/components/system-brand.tsx`
  - Displayed the existing system name as `<current name> / SyncAPI` in the app brand area, while avoiding duplicate `SyncAPI` when already present.
  - Reason: show `SyncAPI` next to the protected `New API`/`NewAPi` branding without replacing the protected identifier.

- `web/default/src/lib/branding.ts`, `web/default/src/features/auth/auth-layout.tsx`, `web/default/src/components/layout/components/public-header.tsx`
  - Added shared display-name formatting and applied it to auth/public top-left brand labels.
  - Reason: cover the top-left brand locations that do not render through `SystemBrand`.

- `web/classic/src/helpers/utils.jsx`, `web/classic/src/components/auth/{LoginForm,RegisterForm,PasswordResetForm,PasswordResetConfirm}.jsx`, `web/classic/src/components/layout/headerbar/HeaderLogo.jsx`
  - Added the same `<current name> / SyncAPI` display formatting for classic auth and header brand labels.
  - Reason: cover deployments that are still serving the classic frontend theme.

- `web/default/src/features/dashboard/index.tsx`
  - Added `SyncAPI` to the dashboard page title while retaining the active dashboard section title.
  - Reason: make the dashboard itself display `SyncAPI` without changing routing or section metadata.

- `common/constants.go`, `setting/system_setting/theme.go`
  - Changed the default frontend theme from `classic` to `default`.
  - Reason: make the redesigned React home page visible when no explicit theme is stored in options.

- `web/default/src/features/home/components/sections/*.tsx`
  - Continued the redesigned home page sections: hero, stats, features, workflow, and CTA.
  - Normalized shadcn/Tailwind usage for button icons, radius, semantic colors, responsive headings, and layout classes.
  - Reason: preserve Gemini's redesign direction while making it compile cleanly and fit project UI conventions.

- `web/default/src/i18n/locales/{en,zh,fr,ja,ru,vi}.json`
  - Added translations for newly introduced home page strings.
  - Reason: prevent untranslated fallback text across supported frontend locales.

## Fixed

- Restored protected `New API` / `QuantumNous` project attribution behavior instead of forcing runtime branding overrides.
- Removed startup-time option overwrites for `SystemName`, `theme`, and `theme.frontend`.
