# v0.0.0

## Changed

- `web/default/src/components/ui/combobox-input.tsx`
  - Rewrote ComboboxInput to properly separate selected value display from search input text. Previously the raw value (e.g., "1") was shown as input text and used as filter, making the channel type selector appear hardcoded to OpenAI. Now shows the selected option's label when closed, and switches to a clean search input when focused.
  - Reason: Channel type dropdown was unusable — displayed numeric IDs instead of provider names and filtered incorrectly.

- `web/default/src/features/channels/components/drawers/channel-mutate-drawer.tsx`
- `web/default/src/features/keys/components/api-keys-mutate-drawer.tsx`
- `web/default/src/features/models/components/drawers/model-mutate-drawer.tsx`
- `web/default/src/features/users/components/users-mutate-drawer.tsx`
- `web/default/src/features/redemption-codes/components/redemptions-mutate-drawer.tsx`
  - Changed submit buttons from `form='xxx' type='submit'` to `onClick={() => form.handleSubmit(onSubmit)()}` with `type='button'`.
  - Reason: The HTML `form` attribute on buttons was not triggering form submission under Base UI's Button component + React 19 Portal rendering. Using programmatic `handleSubmit()` invocation bypasses this issue entirely.

- `service/token_billing_multiplier.go`, `dto/openai_response.go`, `dto/realtime.go`
  - Added a shared one-time 1.2 token billing multiplier with JSON-hidden guard fields for text and realtime usage structs.
  - Reason: align billed NewAPI token counts with upstream OpenAI token accounting drift while preventing repeated multiplication.

- `relay/helper/price.go`, `service/text_quota.go`, `service/quota.go`, `relay/channel/openai/relay-openai.go`, `relay/channel/dify/relay-dify.go`, `relay/channel/xai/text.go`
  - Applied the shared multiplier at pre-consume and settlement boundaries, including tiered billing snapshots, text/audio/realtime usage, and local fallback totals.
  - Reason: ensure every token calculation path is adjusted once and only once before quota charging.

- `service/token_billing_multiplier_test.go`, `relay/helper/price_test.go`, `service/text_quota_test.go`
  - Added multiplier idempotency coverage and updated quota expectations for the 1.2 token adjustment.
  - Reason: lock in the adjusted billing behavior and catch accidental double multiplication.

- `service/channel_affinity_usage_cache_test.go`
  - Replaced timestamp-derived cache keys with test-name plus atomic sequence keys.
  - Reason: avoid cache key collisions on Windows timer resolution during channel-affinity usage cache tests.

- `relay/helper/stream_scanner.go`
  - Preserved a pre-existing `StreamStatus` instead of replacing it at stream scanner startup.
  - Reason: keep previously recorded stream diagnostics and satisfy the existing pre-initialized status contract.

- `web/default/src/components/layout/components/system-brand.tsx`
  - Displayed the existing system name as `<current name> / SyncAPI` in the app brand area, while avoiding duplicate `SyncAPI` when already present.
  - Reason: show `SyncAPI` next to the protected `New API`/`NewAPi` branding without replacing the protected identifier.

- `web/default/src/lib/branding.ts`, `web/default/src/features/auth/auth-layout.tsx`, `web/default/src/components/layout/components/public-header.tsx`
  - Added shared display-name formatting and applied it to auth/public top-left brand labels.
  - Reason: cover the top-left brand locations that do not render through `SystemBrand`.

- `web/classic/src/helpers/utils.jsx`, `web/classic/src/components/auth/{LoginForm,RegisterForm,PasswordResetForm,PasswordResetConfirm}.jsx`, `web/classic/src/components/layout/headerbar/HeaderLogo.jsx`
  - Added the same `<current name> / SyncAPI` display formatting for classic auth and header brand labels.
  - Reason: cover deployments that are still serving the classic frontend theme.

- `web/default/src/features/dashboard/index.tsx`
  - Added `SyncAPI` to the dashboard page title while retaining the active dashboard section title.
  - Reason: make the dashboard itself display `SyncAPI` without changing routing or section metadata.

- `common/constants.go`, `setting/system_setting/theme.go`
  - Changed the default frontend theme from `classic` to `default`.
  - Reason: make the redesigned React home page visible when no explicit theme is stored in options.

- `web/default/src/features/home/components/sections/*.tsx`
  - Continued the redesigned home page sections: hero, stats, features, workflow, and CTA.
  - Normalized shadcn/Tailwind usage for button icons, radius, semantic colors, responsive headings, and layout classes.
  - Reason: preserve Gemini's redesign direction while making it compile cleanly and fit project UI conventions.

- `web/default/src/i18n/locales/{en,zh,fr,ja,ru,vi}.json`
  - Added translations for newly introduced home page strings.
  - Reason: prevent untranslated fallback text across supported frontend locales.
